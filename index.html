HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • It is important to use the heading tags appropriately to make the page accessible to users who use screen readers
  • It is important to include alt text after an image for users using a screen reader or if the link to the image breaks
  • The footer usually contains the author, copyright, contact, sitemap, and navigation.
  • The Header appears at the top of the page and usually contains a navigation menu and/or introductory content.
  • An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.
  • The id attribute is more specific than the class attribute, and the class attribute is more specific than tags

CSS

  • CSS, or cascading style sheets, is a programming language used to define the style of a webpage and how it will be presented to users.
  • We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options
  • Webpages can be styled using inline css, internal css style sheets, or external style sheets.
  • Internal style sheets contain the css rules for that page in the head.
  • The most common way to style a webpage is with an external style sheet, in a seperate file, which is then linked through the head of your html document.
  • CSS uses selectors followed by property-value pairs to make declarations and all given declarations make up a declaration block.
  • Margin determines the space given around an element, padding determines the space given around the content within a given element.
  • The height and width of an element, plus the padding, border and margins on all sides constitute the box model. You can calculate the total height/ width of an element by adding the values of these properties together.
  • You can use a comma to combine selectors and allow for the styles that follow to effect both selectors.
  • You can select colors in CSS by typing the name of the color, entering the hexidecimal value, the rgb or rgba value, or the HSL value you desire.

GIT

  • We use the cd command to navigate to file directories.
  • We use the code . command to open the current directory in vs code
  • The git command must precede every Git statement.
  • The checkout command is to move the working branch to a new branch.
  • The -b flag creates a new branch
  • After the flag, we create the name of our new branch, it is wise to include a prefix, such as feature, to indicate what is being changed in this branch
  • We use the git status command to ensure that we are working in the new branch
  • To add code to the current branch, we use the command git add -A to indicate that we want to add all changes
  • Once we have staged files to be committed, we use the git commit command, followed by a commit message, to commit the changes
  • The commit message should describe what the commit contains
  • We can use the pull command to receive a branch's modifications into the local environment
  • It is important to always do a git pull to make sure that you have the latest version of the project on your local computer before you start to work.
  • We can use the push command to push the changes we have locally to our remote GitHub branch.
  • After pushing the changes to github, we need to navigate to the branch we were working on, use the new pull request button, ensure that the branches are able to be merged, then merge them.
  • The git command cheat sheet can be found here.

Java Script

  • JavaScript is what allows us to interact with webpages.
  • Variables are containers that store values. You use the "var" keyword to declare a variable and then use the name that you will give the variable.
  • Variables may hold different values, including strings, which need to be enclosed in quotation marks, numbers which do not need quotation marks, the words true and false, and arrays which allow you to store multiple values and objects.
  • We can use conditionals to test if an expression returns true or false. A common way to do this is with if else statements, where if the expression is true, we will recieve one result and if false we will recieve the else result.
  • We can use functions to package funtionality we wish to reuse. This allows us to define some code as a function which is executed when we call that function's name instead of rewriting the same code repeatedly.
  • Event handlers allow us to make our sites more interactive by listening for certain activity in the browser and running specific code in response to actions taken by the user.
  • JS code is read from top to bottom, so the order we code statements in will effect the final outcome. This is why we must take into account control flow when writing JS.
  • Control flow can be interrupted by conditonal statements.
  • Arrays are variables which allow us to hold a group of data. We declare the variable in the same way but we assign multiple values instead of just one. Arrays are especially useful when we want to perform a task on multiple pieces of data at once, including different types of data.
  • Pieces of data stored in an array are indexed, beginning with the first value being 0,and subsequent values increasing by one each time.
  • For loops allow a task to be performed on all items in an array by allowing a single block of code to be executed until we use another statement to interrupt the control flow. To use a for loop we need the array's length to know how many times the code can be run until there are no items left in the array.
  • For loops contain three important statements, the starting point for the loop, the condition that dictates if the loop keeps running, and the statement that allows the array to iterate over each item.
  • We use the function keyword to create a function, then we give it a unique name which will be used whenever we call the function. Functions must be called if they are to be executed.
